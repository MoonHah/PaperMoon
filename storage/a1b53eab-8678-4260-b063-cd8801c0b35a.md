# Git 常用 CLI 手册

这是一份日常使用 Git 和 GitHub 的简洁速查表，适合用于 `LLMLearn` 这类学习项目。

## 1. 查看状态

```bash
git status
```

查看当前有哪些文件被修改、新增、删除，以及是否已经暂存。

```bash
git log --oneline
```

查看提交历史。

```bash
git remote -v
```

查看当前仓库绑定的 GitHub 远程地址。

## 2. 初始化仓库

```bash
git init
git branch -M main
```

把当前文件夹初始化为 Git 仓库，并把主分支命名为 `main`。

## 3. 第一次上传到 GitHub

先在 GitHub 创建一个空仓库，然后在本地执行：

```bash
git add .
git commit -m "Initial commit"
git remote add origin https://github.com/你的用户名/仓库名.git
git push -u origin main
```

如果使用 SSH：

```bash
git remote add origin git@github.com:你的用户名/仓库名.git
git push -u origin main
```

## 4. 日常提交

```bash
git status
git add .
git commit -m "描述这次改动"
git push
```

常见例子：

```bash
git commit -m "Add day-03 notes"
git commit -m "Update RAG examples"
git commit -m "Fix uv project setup"
```

## 5. 只提交指定文件

```bash
git add README.md
git add day-03/note.md
git commit -m "Update README and day-03 notes"
git push
```

适合不想一次性提交所有改动的时候。

## 6. 拉取 GitHub 最新代码

```bash
git pull
```

如果你在另一台电脑或 GitHub 网页上改过代码，先 `pull` 再继续写。

## 7. 查看具体改动

```bash
git diff
```

查看还没有暂存的改动。

```bash
git diff --staged
```

查看已经 `git add`、但还没 commit 的改动。

## 8. 撤销改动

撤销某个文件的未暂存改动：

```bash
git restore 文件名
```

例如：

```bash
git restore README.md
```

取消暂存，但保留文件内容：

```bash
git restore --staged 文件名
```

例如：

```bash
git restore --staged README.md
```

## 9. 分支

查看分支：

```bash
git branch
```

创建并切换到新分支：

```bash
git switch -c 分支名
```

例如：

```bash
git switch -c day-04-rag
```

切回主分支：

```bash
git switch main
```

把分支推送到 GitHub：

```bash
git push -u origin 分支名
```

## 10. 删除文件并提交

```bash
git rm 文件名
git commit -m "Remove unused file"
git push
```

如果文件已经被你手动删除：

```bash
git add .
git commit -m "Remove unused file"
git push
```

## 11. 常见问题

如果看到 `fatal: not a git repository`，说明当前目录不是 Git 仓库。先进入项目根目录：

```bash
cd /Users/moonhuang/LLMLearn
```

如果不小心提交了不该提交的文件，先把它加入 `.gitignore`，然后从 Git 追踪中移除：

```bash
git rm --cached 文件名
git commit -m "Stop tracking ignored file"
git push
```

例如不要提交 `.env`：

```bash
git rm --cached .env
git commit -m "Stop tracking env file"
git push
```

## 12. 推荐日常流程

每次开始学习前：

```bash
cd /Users/moonhuang/LLMLearn
git pull
```

学习结束后：

```bash
git status
git add .
git commit -m "Add day-0x learning notes"
git push
```
